# 论文2故事与创新点梳理

> 本文档记录论文2的故事主线、问题域、创新点层级和与 ICUS 论文的边界关系。论文2不是 ICUS 的简单增强版，而是从局部 cooperative gap development 升级到长匝道、多 MV、rolling、密集交通下的 hybrid gap supply 与资源预约重规划问题。

---

## 1. 与 ICUS 论文的关系

两篇论文的关系应写成：

```text
ICUS 论文：
    候选 gap 的局部纵向协作发展、评分选择和轨迹执行。

论文2：
    长匝道、多 MV、rolling 和密集交通下的 gap-supply action 生成、模式选择、资源预约、重规划和交通级验证。
```

不要写成：

```text
论文1 = 简化版
论文2 = 完整版
```

推荐写成：

```text
论文1 = local longitudinal cooperative gap development branch
论文2 = rolling hybrid gap supply and coordination framework
```

ICUS 解决“一个候选 gap 如何以低主线扰动发展并执行”。论文2解决“当主线很密、多 MV 竞争有限主线资源、单纯纵向调整不足时，如何在 rolling horizon 中动态供应、选择和预约合流机会”。

---

## 2. 论文2核心定位

论文2研究一种面向密集匝道合流场景的 rolling hybrid gap supply 方法。其核心不再是单个候选 gap 如何纵向扩张，而是在长时空范围内，为多个 MV 动态生成可执行的 gap-supply action，并在纵向调整、减速等待、主线车向内侧换道释放空间等模式之间进行评价、选择、预约和重规划。

一句话定位：

```text
Gap supply is formulated as a rolling, mode-aware, and resource-constrained coordination problem.
```

论文2的关键词可以是：

```text
rolling hybrid gap supply
mode-aware gap-supply valuation
multi-MV reservation and replanning
lane-change-assisted gap supply
resource-constrained on-ramp merging
mixed CAV/HDV merging
SUMO closed-loop validation
```

---

## 3. 论文2核心矛盾

从论文1到论文2的问题升级：

```text
论文1解决：
    一个候选 gap 如何以低主线扰动发展并执行。

论文2解决：
    当主线很密、多 MV 竞争有限主线资源、单纯纵向调整不足时，如何在 rolling horizon 中动态供应、选择和预约合流机会。
```

论文2的新矛盾包括：

```text
1. 主线很密，目标车道相邻 gap 余量不足，单纯纵向调整无法构造安全 gap。

2. 多个 MV 会竞争同一个 gap、同一组协作车辆或同一个合流时间窗。

3. 合流机会不再是静态候选 gap，而是 gap、mode、time window、helper vehicles 和 reservation 的组合 action。

4. CAV 渗透率不足或 HDV 响应不确定时，gap supply 成功率下降。

5. 纯数值仿真不足以证明交通级扰动传播，需要 random traffic 和 SUMO closed-loop 验证。
```

## 4. 论文2方法总流程

建议方法链：

```text
1. Rolling perception and prediction
2. Gap-supply action set construction
3. Hybrid gap supply mode generation
4. Mode-specific feasibility and cost evaluation
5. Mode-aware gap-supply valuation
6. Multi-MV reservation and replanning
7. Closed-loop validation
```

其中 action 不再只是一个 gap，而是：

```text
a = {
    MV id,
    target gap,
    supply mode,
    helper vehicles,
    receiving lane gap if lane change is involved,
    merge time window,
    reserved resource interval,
    expected disturbance cost,
    expected success probability
}
```

## 6. 贡献点 1：Rolling Gap-Supply Action Set Construction

论文2中的 action set 不应被写成普通 gap 筛选，而是 rolling action generation。

### 6.1 贡献内容

在 rolling horizon 中，根据 MV 运动学约束、candidate gap 时空位置、协作模式可行性和资源预约状态，生成每个 MV 在当前 horizon 内可执行的 gap-supply action。

action 应包含：

```text
MV id
target gap
supply mode
helper vehicles
merge time window
reserved resource interval
expected disturbance cost
expected success probability
```

### 6.2 推荐贡献表述

```text
We develop a rolling gap-supply action construction method that generates executable merging opportunities for each MV by jointly considering gap spatiotemporal availability, MV kinematics, supply-mode feasibility, and resource reservation status.
```

中文：

```text
提出一种面向长匝道 rolling 场景的 gap-supply action 构造方法，根据 MV 运动学约束、候选 gap 时空位置、协作模式可行性和资源预约状态，生成每个 MV 在当前 rolling horizon 内可执行的合流机会。
```

## 7. 贡献点 2：Hybrid Gap Supply Modes

论文2要从论文1的纵向 gap development 扩展为多模式 gap supply。

### Mode 1: Longitudinal Gap Development

继承 ICUS 方法作为纵向发展 branch：

```text
boundary vehicles adjust longitudinally
target gap expands under adjacent-gap safety constraints
disturbance cost inherited from ICUS model
```

### Mode 2: Defer / Waiting

当当前 gap supply 代价过高或不可行时，MV 等待后续窗口：

```text
defer merge
preserve current lane
re-evaluate in next rolling horizon
```

### Mode 3: Lane-Change-Assisted Gap Supply

当目标车道过密且纵向余量不足时，协调目标车道主线车向内侧车道换道，释放目标车道空间：

```text
select lane-changing helper vehicle
check inner-lane receiving gap
estimate lane-change time and disturbance
estimate released target-lane space
update target gap supply
```

### Mode 4: Hybrid Supply

组合轻微纵向调整、换道释放和 rolling 重规划：

```text
minor longitudinal adjustment
lane-change-assisted release
defer if necessary
reservation update
```

### 推荐贡献表述

```text
We propose a hybrid gap supply framework that extends local longitudinal gap development to multiple supply modes, including longitudinal adjustment, defer/waiting, lane-change-assisted space release, and hybrid supply under rolling replanning.
```

## 8. 贡献点 3：Mode-Aware Gap-Supply Valuation

不同 supply mode 的代价结构不同，因此不能只用论文1的 `C_coop + C_ego`。论文2需要 mode-aware 评价。

### 8.1 推荐路线

```text
分模式生成候选 action
分模式求解可行性和代价
统一评分比较
rolling 中选择和预约
```

默认建议：

```text
先采用“分模式求解 + 统一评价 + rolling 预约重规划”，不要一开始强行做全局联合优化。
```

### 8.2 评分结构草案

```text
J(a) =
    w_m C_merge
  + w_l C_longitudinal
  + w_c C_lane_change
  + w_i C_inner_lane_impact
  + w_r C_reservation
  + w_u C_uncertainty
```

其中：

```text
C_merge             MV 合流效率或自车控制代价
C_longitudinal      目标车道纵向协作扰动
C_lane_change       换道车辆换道代价
C_inner_lane_impact 内侧车道受扰动代价
C_reservation       资源占用和冲突代价
C_uncertainty       HDV 不服从、预测误差或执行失败风险
```

### 8.3 Lane-Change-Assisted Action 需要考虑

```text
内侧车道接收 gap 是否可用；
换道车辆的换道时间和安全约束；
目标车道释放出的空间收益；
内侧车道扰动代价；
换道失败或 HDV 不服从风险。
```

### 8.4 推荐贡献表述

```text
We construct a mode-aware gap-supply valuation mechanism that compares actions across longitudinal adjustment, waiting, and lane-change-assisted supply by jointly considering MV merging efficiency, target-lane disturbance, lane-changing disturbance, inner-lane impact, resource occupancy, and execution uncertainty.
```

## 9. 贡献点 4：Multi-MV Rolling Reservation and Replanning

论文2需要处理多个 MV 对有限 gap-supply resources 的竞争。

### 9.1 资源对象

```text
target gap
boundary helper vehicles
lane-change helper vehicle
inner-lane receiving gap
merge time window
road segment occupancy
```

### 9.2 机制

```text
1. action scoring
2. resource reservation
3. conflict detection
4. reservation release if action fails or becomes dominated
5. rolling replanning in next horizon
```

### 9.3 推荐贡献表述

```text
We propose a multi-MV rolling reservation and replanning mechanism that treats future gaps, helper vehicles, inner-lane receiving gaps, and merging time windows as shared resources, reducing conflicts among MVs competing for the same mainline opportunity.
```

中文：

```text
提出一种多 MV rolling 资源预约与重规划机制，将未来 gap、协作车辆、内侧车道接收空间和合流时间窗作为可竞争资源进行动态预约、释放和更新，以减少多个 MV 对同一主线资源的冲突。
```

## 10. 贡献点 5：Traffic-Level and Mixed-Traffic Validation

论文2的实验验证应明显强于 ICUS，因为它处理 rolling、多车、换道和不确定性。

建议分层：

```text
Level 1: fixed rolling scenarios
Level 2: random traffic numerical simulation
Level 3: SUMO closed-loop
Level 4 optional: CAV/HDV mixed traffic
```

### 10.1 HDV/CAV 混行处理原则

如果 HDV 只作为实验变量，可以写成验证增强：

```text
evaluate success rate, disturbance, and efficiency under different CAV penetration rates and HDV compliance probabilities
```

如果 HDV 服从概率进入 action feasibility 或 expected cost，则应提升为正式方法贡献：

```text
expected success probability
expected disturbance cost
uncertainty-aware action valuation
```

## 11. 论文2实验链

建议实验顺序：

```text
1. Rolling action set validity
   验证 action generation 能覆盖可执行合流机会，并排除明显不可执行 action。

2. Dense-mainline failure of longitudinal-only branch
   证明主线很密时，仅靠 ICUS 的纵向 branch 会出现 gap supply 不足。

3. Lane-change-assisted gap supply
   验证内侧换道释放目标车道空间能提高 gap supply success rate。

4. Mode-aware valuation
   对比 longitudinal-only、waiting-only、lane-change-first、proposed mode-aware valuation。

5. Multi-MV reservation
   验证 reservation/replanning 能减少多个 MV 对同一 gap、helper vehicle 或 time window 的冲突。

6. Random traffic statistics
   在随机车流下统计 success rate、delay、mainline disturbance、throughput。

7. SUMO closed-loop validation
   用更真实车辆模型和交通反馈验证扰动传播、换道执行和闭环性能。

8. CAV/HDV mixed traffic
   分析不同 CAV 渗透率和 HDV 响应不确定性下的 gap supply 成功率、主线扰动和合流效率。
```

## 12. 当前待定问题

论文2仍需进一步确定：

```text
1. lane-change-assisted action 的可行性约束；
2. 内侧车道接收 gap 的安全模型；
3. 换道释放空间收益如何量化；
4. 是否采用分模式求解 + 统一评分，还是构造联合优化；
5. HDV 服从概率是否进入 action expected cost；
6. rolling reservation 的冲突检测和释放机制；
7. SUMO 中车辆模型、lane-change model 和控制接口如何闭环。
```

默认建议：

```text
先采用“分模式求解 + 统一评价 + rolling 预约重规划”，不要一开始强行做全局联合优化。
```

原因：

```text
1. 分模式求解更容易继承 ICUS 的 longitudinal branch；
2. lane-change-assisted branch 的可行性与代价模型仍在设计中；
3. 统一评价能先支撑实验比较；
4. rolling 预约重规划可以自然处理 action 失效和交通状态变化；
5. 全局联合优化可以作为后续理论增强，而不必成为论文2第一版的负担。
```

## 13. 论文2与 ICUS 的边界总结

| 维度 | ICUS 论文 | 论文2 |
|---|---|---|
| 问题对象 | 单次决策中的 candidate gap development | rolling horizon 中的 gap-supply action |
| 核心模式 | 纵向边界车辆协作 | 纵向发展、等待、换道辅助、混合模式 |
| 车辆规模 | 单 MV 或简单局部场景 | 多 MV 资源竞争 |
| 主线状态 | 可用来验证局部扰动模型的确定性场景 | 密集、随机、动态变化交通 |
| 验证方式 | 数值验证和消融 | random traffic + SUMO closed-loop + optional CAV/HDV |
| 核心贡献 | minimum-disturbance cooperative gap development and valuation | rolling hybrid gap supply and coordination |

论文2的写法不能是：

```text
ICUS + 换道 + SUMO
```

而应是：

```text
rolling hybrid gap supply and coordination under dense on-ramp merging
```
